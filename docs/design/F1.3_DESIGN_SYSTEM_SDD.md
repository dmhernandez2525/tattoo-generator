## 1. Overview

### Purpose
Define a cohesive design system for INK SYNTHESIS including brand identity, reusable components, motion, responsive layouts, dark/light modes, and loading states.

### Goals
- Establish visual tokens and typography for a dark, artistic theme.
- Provide reusable UI primitives with consistent styling.
- Support dark/light mode switching without visual regressions.
- Implement loading, empty, and error states across feature surfaces.

### Non-Goals
- Marketing site content strategy (handled in Phase 10).
- Advanced illustration system or 3D assets.

## 2. Design Tokens

### Color System
- Base: `bg-dark`, `bg-surface`, `text-primary`, `text-muted`.
- Accents: `neon-cyan`, `neon-purple`, `accent-gold` for calls to action.
- Status: `success`, `warning`, `error`, `info`.

### Typography
- Display: `Orbitron` for headings.
- Body: `Inter` for paragraphs and UI labels.

### Spacing & Radius
- Spacing scale aligned to Tailwind defaults.
- Radius: `6px` (default), `10px` (cards), `9999px` (chips).

## 3. Component Library

### Base Components
- `Button`, `Input`, `Card`, `Slider`, `Badge`, `Tabs`, `Dialog`.
- Variants defined with `class-variance-authority`.

### Layout Components
- `PageShell` for consistent margins and max-width.
- `SectionHeader` for title + description patterns.
- `EmptyState`, `ErrorState`, `LoadingState` components.

### Motion Guidelines
- Page transitions: subtle fade/scale.
- Action feedback: hover glow + slight scale.
- Loading: skeleton shimmer + progress bars for generation.

## 4. Theme Management

### Approach
- Use a CSS variable-driven theme with `data-theme` on `<html>`.
- `ThemeProvider` toggles between `dark` and `light`.
- Persist theme in `localStorage` with system preference fallback.

### Tokens by Theme
- Dark: neon accents, deep backgrounds.
- Light: soft neutral surfaces, toned-down accents.

## 5. Accessibility

- Ensure minimum 4.5:1 contrast on text.
- Focus rings visible on keyboard navigation.
- Animations respect `prefers-reduced-motion`.

## 6. Testing & Validation

- Visual regression checks on core components.
- Component tests for variant rendering.
- Manual QA for dark/light toggle.

## 7. Deliverables

- `src/styles/theme.css` with token definitions.
- `ThemeProvider` for runtime switching.
- Updated UI primitives in `src/components/ui`.
- Documentation in `docs/design/` for usage patterns.
